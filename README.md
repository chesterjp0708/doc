# doc
AI活用による単体テスト作成効率化 PoC提案書

1. 背景

近年、生成AI技術の急速な発展により、ソフトウェア開発工程への活用が進んでいる。

特に、設計書解析、テストケース作成、コードレビューなどの工程においては、AIによる生産性向上および品質向上が期待されている。

一方で、AIによる誤生成（Hallucination）、機密情報の取り扱い、成果物品質の担保など、実運用に向けた課題も存在する。

そのため本提案では、単体テスト作成工程を対象としたPoC（Proof of Concept）を実施し、AI活用による効果および運用上の課題を検証する。

2. 目的

本PoCでは、以下の観点について検証を行う。

2.1 業務効率向上

単体テスト作成工数の削減

設計書解析作業の効率化

テスト作成プロセスの標準化

2.2 品質向上

設計漏れの早期発見

テスト観点の網羅性向上

設計レビュー品質の向上

2.3 AI活用ノウハウの蓄積

AI利用ガイドライン整備

AI活用パターンの確立

将来的な適用範囲拡大に向けた知見獲得

5. Phase1（PoC対象）

5.1 Agent1：設計書解析・UT項目生成 Agent

概要

Excel形式の詳細設計書を解析し、単体テスト作成に必要な情報を整理する。

AIへ直接Excelファイルを入力するのではなく、必要情報を抽出してMarkdown形式へ変換し、その内容をGitHub Copilotへ入力することで、テスト項目作成を支援する。

6. セキュリティおよび運用方針

AI利用にあたり、機密情報保護の観点から以下のルールを適用する。

6.1 入力データ制限

顧客情報を投入しない

個人情報を投入しない

本番データを利用しない

必要最小限の情報のみ利用する

6.2 運用ルール

AI投入前に内容を確認する

AI生成結果をそのまま採用しない

成果物は必ず人間がレビューする

最終判断は開発者が実施する
